# Паттерн Memento (Хранитель) в Python

## 1. Что это и зачем нужен?

**Паттерн Memento** (или «Хранитель») — это способ сохранить текущее состояние объекта, чтобы потом можно было легко «откатить» его назад.

**Главная идея:** Не трогать внутренности объекта, а аккуратно упаковать его состояние в специальную «капсулу» (снимок) и хранить её отдельно.

**Где пригодится:**
- Кнопка «Отменить» (Ctrl+Z) в текстовом редакторе.
- Сохранение игры (вы сохраняетесь, а потом загружаетесь с того же места).
- История изменений в графическом редакторе.

## 2. Аналогия: сохранение в видеоигре

Представь сложную видеоигру с открытым миром.

- **Игрок (Originator)** — это персонаж в игре. У него есть уровень здоровья, количество монет, позиция на карте, надетые доспехи и прогресс по квестам. В любой момент он может измениться: потерять здоровье в бою, найти золото, переместиться в другой город.

- **Сохранение (Memento)** — это файл сохранения на компьютере или в облаке. В этом файле записаны все важные параметры персонажа. Сам файл игрок не открывает и не редактирует — он просто знает, что если загрузить его, персонаж вернётся ровно в то состояние, которое было на момент сохранения.

- **Система сохранений (Caretaker)** — это меню игры, где отображается список сохранений. Она умеет создавать новые файлы («Сохранить игру») и загружать старые («Загрузить игру»). Система не знает, что именно означает число 42 в поле «здоровье» — она просто хранит файлы как есть и передаёт их обратно игре по запросу.

### Как это соотносится с паттерном:

| В игре | В паттерне |
|--------|-------------|
| Персонаж (с его здоровьем, инвентарём, позицией) | Originator |
| Файл сохранения (сериализованные данные) | Memento |
| Меню слотов сохранения / автобэкапы | Caretaker |

### Процесс шаг за шагом:

1. Ты подходишь к опасному боссу и делаешь **«Сохранить»**. Игра (Caretaker) просит персонажа (Originator) выдать снимок его состояния. Персонаж собирает все свои параметры и отдаёт их в виде файла сохранения (Memento). Caretaker кладёт этот файл в первый слот.

2. Ты проигрываешь боссу — здоровье упало до нуля, часть монет потеряна. Персонаж изменился.

3. Тебе жаль потерянного прогресса, ты выбираешь **«Загрузить»** первый слот. Caretaker достаёт сохранённый файл и передаёт его персонажу. Персонаж восстанавливает из файла все параметры: здоровье, монеты, позицию — ты снова стоишь перед боссом, как будто ничего не случилось.


Именно так и работает паттерн Memento в программировании — как надёжная система сохранений для любого объекта.

## 3. Три главных героя паттерна

| Роль | Обязанность | Пример в Python |
|------|-------------|----------------|
| **Originator** | Создаёт снимки и восстанавливается из них. Умеет менять своё состояние. | Класс `TextEditor` |
| **Memento** | Хранит один снимок состояния. Обычно это неизменяемый объект (кортеж, строка, `namedtuple`). | Объект с полем `_state` |
| **Caretaker** | Просит снимок у Originator'а, хранит его, но **не изменяет** его содержимое. | Список `history` в главной программе |

## 4. Пример «Игрок с сохранением»

### Пример 4.1 — без паттерна (как НЕ надо делать)

```python
class Player:
    def __init__(self, health, level):
        self.health = health
        self.level = level

# Сохраняем «в лоб»
player = Player(100, 1)
save_health = player.health
save_level = player.level

# Игрок изменился
player.health = 50

# Восстанавливаем
player.health = save_health
player.level = save_level
```

**Проблема:** Если у объекта 10 полей, придётся вручную сохранять каждое. А если добавится новое поле — код сломается. Нарушается инкапсуляция.

### Пример 4.2 — с паттерном Memento

```python
# 1. Memento — просто контейнер для данных
class Memento:
    def __init__(self, health, level):
        self._health = health
        self._level = level

    def get_health(self):
        return self._health

    def get_level(self):
        return self._level

# 2. Originator — игрок, который умеет сохраняться и восстанавливаться
class Player:
    def __init__(self, health, level):
        self.health = health
        self.level = level

    def save(self) -> Memento:
        """Создаёт снимок текущего состояния"""
        return Memento(self.health, self.level)

    def restore(self, memento: Memento):
        """Восстанавливает состояние из снимка"""
        self.health = memento.get_health()
        self.level = memento.get_level()

    def __str__(self):
        return f"Player(health={self.health}, level={self.level})"

# 3. Caretaker — кто-то, кто хранит историю
class History:
    def __init__(self):
        self._saves = []   # список снимков

    def push(self, memento: Memento):
        self._saves.append(memento)

    def pop(self) -> Memento:
        return self._saves.pop()

# Использование
player = Player(100, 1)
history = History()

print("Начало:", player)
history.push(player.save())   # сохранили

# Игрок получает урон и качается
player.health = 80
player.level = 2
print("После изменений:", player)

# Откат
player.restore(history.pop())
print("После отката:", player)

```

**Вывод:**

Начало: Player(health=100, level=1)

После изменений: Player(health=80, level=2)

После отката: Player(health=100, level=1)

## 5. Когда использовать, а когда нет

### Хорошо подходит:
- Нужна кнопка Undo/Redo.
- Нужно сохранять промежуточные состояния (чекпоинты).
- Класс с большим количеством полей, и не хочется писать кучу кода для их сохранения.

### Не стоит использовать:
- Объект содержит очень много данных (каждый снимок съедает память). Решение — хранить не полную копию, а только изменения (паттерн Command).
- Состояние меняется очень часто (каждый клик мыши — новый снимок).
- Задача решается проще через `copy.deepcopy()` и список копий (но тогда теряется контроль).

## 6. Итог

**Паттерн Memento** — это три компонента:
1. **Originator** (создаёт снимки и восстанавливается)
2. **Memento** (неизменяемый контейнер с состоянием)
3. **Caretaker** (хранит снимки, не вникая в их содержание)

В Python Memento часто делают через `namedtuple`, простые переменные или `dataclass(frozen=True)`, потому что не нужно плодить лишние классы. Главное — **разделить ответственность**: объект сам знает, как сделать снимок, а кто-то другой (Caretaker) решает, когда и сколько снимков хранить.

Теперь, когда вы нажимаете Ctrl+Z, вы знаете, что, скорее всего, внутри работает Memento.
